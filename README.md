| **Tema escolhido** | Calculadora Avançada |
| **Elementos do grupo** | Guilherme Carvalho — a22405584 |
| **Descrição geral do projeto** | Aplicação em Java com interface gráfica (Swing) que funciona como uma calculadora avançada. Permite realizar operações matemáticas básicas de forma determinística (sem recurso a LLM) e operações avançadas, como derivação simbólica, simplificação de expressões, resolução de equações e interpretação de texto matemático em linguagem natural, delegando essas tarefas a um LLM. |
| **Funcionalidade Java (não-LLM)** | Implementação das operações matemáticas determinísticas: soma (+), subtração (−), multiplicação (*), divisão (/), potência (^) e raiz, executadas localmente em Java para maior eficiência e previsibilidade. |
| **Funcionalidade delegada ao LLM** | Processamento de operações matemáticas avançadas difíceis de implementar manualmente em Java, nomeadamente: derivação simbólica, simplificação de expressões, resolução de equações simples, interpretação de texto matemático escrito por extenso e geração de uma frase criativa natalícia contendo o resultado (Modo Natal). |
| **Classes principais** | `Main`, `Calculadora` (operações simples em Java), `MotorCalculo` (decisão entre Java ou LLM), `LLMCalculadora` (operações avançadas via LLM), `CalculadoraSwing` (interface gráfica), `Utils`, `JSONUtils`, `Comandos` (interface de terminal para apresentação do menu). |
| **Fluxo de dados (resumo)** | Utilizador → Interface gráfica (Swing) → Motor de cálculo → (decisão: operação simples em Java **ou** operação avançada no LLM) → Execução → Resultado formatado → Histórico |
| **Prompts de integração esperadas (planeamento)** | Inicialmente foi considerado o uso de respostas estruturadas em JSON para todas as operações avançadas, de forma a facilitar o parsing e a validação dos resultados devolvidos pelo modelo. |
| **Prompts de integração usadas (implementação final)** | **1 — Derivação:** “Deriva a seguinte expressão matemática: {expressão}. Responde APENAS com a derivada final. Não expliques nada. Não uses JSON. Não escrevas mais nenhuma palavra.”<br><br> **2 — Simplificação:** “Simplifica a seguinte expressão matemática: {expressão}. Responde APENAS com a expressão simplificada. Não expliques nada. Não uses JSON. Não escrevas mais nenhuma palavra.”<br><br> **3 — Resolução de equações:** “Resolve a seguinte equação matemática: {equação}. Responde APENAS com o resultado final. Não expliques nada. Não uses JSON.”<br><br> **4 — Interpretação de texto livre:** “Interpreta o texto seguinte como um problema matemático e calcula o resultado final. Responde APENAS com o resultado final. Não expliques nada. Não uses JSON. Não escrevas mais nenhuma palavra.”<br><br> **5 — Modo Natal:** “Gera UMA única linha curta de música natalícia divertida. A linha DEVE incluir exatamente o valor {resultado}. Não expliques nada. Não uses JSON. Responde apenas com a frase.” |
| **Funções desenvolvidas sem recurso a LLM** | Todas as funcionalidades básicas foram implementadas sem recurso a LLM, incluindo as operações matemáticas fundamentais e o motor de cálculo. A classe `LLMCalculadora` foi criada pelo autor, tendo apenas sofrido pequenos ajustes com apoio de um LLM, nomeadamente na afinação das prompts. As restantes componentes do projeto contaram com auxílio pontual de um LLM para melhorias de organização, clareza e usabilidade. |
